Tree

The Tree concept contains a few requirements that are common to all tree concepts. These include the ability to obtain the root vertex and to iterate through child vertices of any given vertex.

Refinement of

IncidenceGraph

Notation

T A type that is a model of Tree.
t An object of type T.
u are objects of type boost::graph_traits<T>::vertex_descriptor.
cv_iter is an object of type boost::tree_traits<T>::child_vertex_iterator.

Associated Types

boost::tree_traits<T>::child_vertex_iterator
A child-vertex iterator pointing to a unique vertex in an abstract tree instance. A child-vertex iterator must be Forward Iterator, and Input Iterator.

Valid Expressions

get_root_vertex(t) Gets the root vertex from the tree.
Return type: vertex_descriptor
child_vertices(u, t) Gets the range of iterators to the children of u from the tree.
Return type: std::pair<child_vertex_iterator, child_vertex_iterator>
Precondition: u is a vertex in the tree.

Complexity Guarantees

  • All operations are O(1).

Models

  • linked_tree
  • bfl_d_ary_tree
  • vebl_d_ary_tree

Concept Checking Class

  
  template <typename T>
  struct TreeConcept {
    typename graph_traits<T>::vertex_descriptor u;
    typename tree_traits<T>::child_vertex_iterator cv_it, cv_it_end;
    T t;
    
    BOOST_CONCEPT_ASSERT((IncidenceGraphConcept<T>));
    
    BOOST_CONCEPT_USAGE(TreeConcept) 
    {
      u = get_root_vertex(t);
      tie(cv_it, cv_it_end) = child_vertices(u,t);
    };
    
  };
