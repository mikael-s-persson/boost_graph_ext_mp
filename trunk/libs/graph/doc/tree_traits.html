tree_traits<Tree>

Trees being a special kind of graph will have all the graph-traits, along with a few specialized traits. This page describes those additional traits that trees have, and which are used in some tree concepts. Given that graph-traits are already very rich in terms of associated types, there is very little need for more. The only extra associated type that tree-traits provide is the child_vertex_iterator, which is conceptually equivalent to a graph's adjacency_iterator. When implementing a tree class that fullfils one or more graph or tree concepts, for associated types that are not required by the concepts, it is ok to use void as the type (when using nested typedefs inside the graph/tree class), or to leave the typedef out of the graph_traits and tree_traits specialization for the graph/tree class. Note that because of the use of traits classes rather than member types, it is not safe (and often will not work) to define subclasses of BGL graph/tree types; those types may be missing important traits and properties that were defined externally to the class definition. 
  template <typename Tree>
  struct tree_traits {
    
    typedef typename Tree::child_vertex_iterator  child_vertex_iterator;
    
    // or, if Tree::child_vertex_iterator does not exist, then this is used:
    // typedef typename graph_traits<Tree>::adjacency_iterator  child_vertex_iterator;
  };
