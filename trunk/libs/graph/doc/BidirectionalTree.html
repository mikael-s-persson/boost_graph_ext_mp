BidirectionalTree

The BidirectionalTree concept contains a few requirements that are common to trees that have a "parent pointer", i.e., that can be traversed in a child-to-parent direction.

Refinement of

Tree

Notation

T A type that is a model of BidirectionalTree.
t An object of type T.
u are objects of type boost::graph_traits<T>::vertex_descriptor.

Valid Expressions

parent_vertex(u, t) Gets the parent of the vertex u from the tree t. If the vertex does not have a parent (i.e., the root), then the return value is equal to graph_traits<T>::null_vertex().
Return type: vertex_descriptor

Complexity Guarantees

  • All operations are O(1).

Models

  • linked_tree (if DirectedS is bidirectionalS)
  • bfl_d_ary_tree
  • vebl_d_ary_tree

Concept Checking Class


  template <typename T>
  struct BidirectionalTreeConcept {
    typename graph_traits<T>::vertex_descriptor u;
    T t;
    
    BOOST_CONCEPT_ASSERT((TreeConcept<T>));
    
    BOOST_CONCEPT_USAGE(BidirectionalTreeConcept) 
    {
      u = parent_vertex(u, t);
    };
    
  };
